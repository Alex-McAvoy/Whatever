### 如何参与

项目组全体成员十分欢迎您为本项目贡献页面

大致流程如下：

1. 将主仓库 Fork 到自己的仓库中
2. 将 Fork 后的分支仓库 Clone 到本地
3. 在本地进行修改后 Commit 更改
4. 将这些更改 Push 到克隆的分支仓库
5. 提交 Pull Request 至主仓库

对于 Commit 摘要，按照如下格式书写：

```bash
commit message = [subject] message
```

常见的 subject 种类以及含义如下：

- `Feat`: 提交新功能
- `Fix`: 修复 bug
- `Docs`: 文档相关的修改
- `Style`: 格式化、标点符号、空白等不影响代码运行的代码风格变更（不影响运行逻辑）
- `Refactor`: 对页面、模块重构（既不是新增功能也不是修复bug的更改，影响运行逻辑）
- `Perf`: 提升性能的修改
- `Test`: 测试相关的内容
- `Chore`: 杂项（构建过程、辅助工具、依赖库等相关内容）
- `Build`: 影响构建系统、外部依赖性的变更
- `CI`: 配置文件、脚本
- `Revert`: 回滚
- `Disuse`：移除废弃代码
